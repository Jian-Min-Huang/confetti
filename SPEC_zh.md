# Confetti.app — 設計規格書

## 1. 簡介

Confetti.app 是一個 macOS 原生應用程式，以透明覆蓋層的形式在全螢幕顯示粒子動畫。設計上透過命令列 `open Confetti.app` 啟動。

## 2. 需求

### 2.1 功能需求

| ID    | 需求                                                                            |
| ----- | ------------------------------------------------------------------------------- |
| FR-1  | 透過 `open Confetti.app` 啟動，無需額外設定                                     |
| FR-2  | 接受 `--style <name>` 參數選擇特效（`confetti`、`falling-leaves`、`fireworks`） |
| FR-3  | 未提供風格參數或參數無效時，預設使用 `confetti`                                 |
| FR-4  | 在所有視窗之上顯示全螢幕透明覆蓋層                                              |
| FR-5  | 覆蓋層必須可穿透點擊——所有滑鼠事件傳遞至底層視窗                                |
| FR-6  | 支援多螢幕——每個螢幕各自播放動畫                                                |
| FR-7  | 接受 `--duration <time>` 參數決定特效播放時間                                   |
| FR-8  | 未提供播放時間參數或參數無效時，預設設定為 `5` 秒                               |
| FR-9  | 動畫結束後自動終止                                                              |
| FR-10 | 不顯示 Dock 圖示，不顯示選單列                                                  |
| FR-11 | 使用多種 Emoji 來 Render 動畫的粒子，先以 `🎉⚽❤️` 作為範例                     |

### 2.2 非功能需求

| ID    | 需求                                  |
| ----- | ------------------------------------- |
| NFR-1 | macOS 13（Ventura）或更新版本         |
| NFR-2 | 零外部依賴                            |
| NFR-3 | GPU 加速渲染，確保動畫流暢            |
| NFR-4 | 自包含二進位——不需要圖片資源檔案      |
| NFR-5 | 完全從命令列建置（不需要 Xcode 專案） |
| NFR-6 | 持續擴充特效的架構設計                |
